# GAP CATEGORIES TO DETECT
Analyze the inputs and identify any gaps or risks in the following categories:

1. Missing Prerequisites:
   - Required properties that do not exist yet (this is normal but should be flagged as LOW severity).
   - Required integrations that are not connected (e.g., Slack integration needed but not set up) -> HIGH severity, blocks execution.
   - Required pipelines or objects that do not exist -> HIGH severity, blocks execution.

2. Live Data Quality Issues (from Stage 4):
   - High null rate on a property used as a trigger source or dynamically referenced (e.g. 22% of contacts have no value for a critical trigger or assignee field) -> MEDIUM severity. Mention how many records will be bypassed.
   - Categorical distribution problems (e.g., all records have the same value in a field meant to drive conditional branching).

3. Graph-Flagged Gotchas & Risks:
   - Match any of the provided product knowledge gotchas to the architecture decision. E.g., if a workflow has re-enrollment enabled and updates properties, check if the `workflow_re_enrollment_behavior` gotcha applies.

   - If tracking contact touches, check if `notes_last_contacted_population` gotcha applies.

4. Unresolved Ambiguities:
   - Any missing thresholds, notification preferences, templates, or dynamic routing properties that are not fully specified.

5. Logical/Configuration Risks:
   - Workflows running on all records instead of filtering by lifecycle stage.
   - Filter conditions that can never trigger (e.g., scoring criteria that are mathematically impossible based on current distributions).
